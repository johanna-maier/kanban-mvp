4. DOCKER SETUP FIX
   - scripts/start-linux.sh and scripts/stop-linux.sh used "sudo docker".
   - User is already in the docker group, so sudo is unnecessary and blocked
     in non-interactive shells.
   - Removed "sudo" from both Linux scripts.

5. RAN ALL TESTS (Docker)
   - Built and started container: bash scripts/start-linux.sh
   - Ran e2e tests against container on port 8000.
   - Stopped container: bash scripts/stop-linux.sh

6. DUAL-MODE PLAYWRIGHT CONFIG
   - Insight: both dev-mode and Docker-mode e2e tests are valuable — they test
     different things.
   - Added DOCKER_TEST env var support to frontend/playwright.config.ts:
       npm run test:e2e               → dev mode (starts next dev, proxies to :8000)
       DOCKER_TEST=1 npm run test:e2e → Docker mode (hits container directly on :8000)
   - When DOCKER_TEST is set, webServer is undefined (Playwright skips launching
     next dev and assumes the container is already running).

7. UPDATED docs/PLAN.md
   - Added a "Test modes" section after "Testing best practices" documenting
     the three test modes: unit tests, e2e dev mode, and e2e Docker mode,
     with commands and guidance on when to use each.
